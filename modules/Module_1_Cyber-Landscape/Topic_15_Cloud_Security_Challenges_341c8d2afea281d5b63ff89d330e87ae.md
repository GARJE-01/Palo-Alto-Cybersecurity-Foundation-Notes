# Topic 15: Cloud Security Challenges

## 🧠 Simple Explanation

Cloud introduces **trade-offs between:**

- Simplicity vs. Security
- Efficiency vs. Visibility
- Agility vs. Control

---

## 🌍 Real-World Example

Fast deployment = Quick launch but misconfigured security → breach risk

---

## ❓ Why It Matters

**Cloud does NOT automatically remove security risks** — it changes them.

---

## 🔑 Key Terms

| Term | Definition |
| --- | --- |
| **Misconfiguration** | Incorrect security settings (biggest cloud risk) |
| **Cloud Sprawl** | Unmanaged cloud resources/costs |
| **Shared Responsibility** | Provider and customer both accountable |

---

## 💡 Exam & Interview Points

- Interview question: *"Does cloud eliminate security risks?"* → **NO**
- **Biggest cloud risk: Misconfiguration** (not hacking)
- Insight: Many breaches happen due to open S3 buckets, exposed APIs

---

## 🚀 Quick Revision

- ▪ Cloud = new risks, not zero risks
- ▪ Misconfiguration = biggest issue
- ▪ Shared responsibility requires both parties to act
