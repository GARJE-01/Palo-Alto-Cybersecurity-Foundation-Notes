# Topic 16: Zero Trust in Cloud

## 📌 Definition

Security model based on "Never Trust, Always Verify" — every user and device must prove authorization.

---

## 🧠 Simple Explanation

**Zero Trust principles:**

- No automatic trust (even inside network)
- Verify every access request
- Assume breach mentality
- Continuous monitoring

---

## 🌍 Real-World Example

Even an employee logging in from office requires:

- Password verification
- MFA (Multi-Factor Authentication)
- Device verification
- Access token validation

---

## ❓ Why It Matters

**Internal users can also be attackers** — traditional perimeter security is obsolete.

---

## 🔑 Key Terms

| Term | Definition |
| --- | --- |
| **Zero Trust** | Never trust, always verify architecture |
| **MFA** | Multi-Factor Authentication |
| **Continuous Authentication** | Ongoing verification throughout session |

---

## 💡 Exam & Interview Points

- Trending interview topic: *"Explain Zero Trust model"*
- Key insight: Replaces traditional "trust inside network" approach
- Relevant for cloud and remote work environments

---

## 🚀 Quick Revision

- ▪ Trust no one, not even employees
