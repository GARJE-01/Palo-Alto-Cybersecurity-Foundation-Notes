# Topic 2: Zero Trust Principles

> Three Core Pillars of Zero Trust Implementation
> 

---

## 📖 CORE IDEA

**Zero Trust is built on 3 main principles**

👉 Not just a concept

👉 Framework with specific rules

👉 Each principle is critical

---

## 🔴 PRINCIPLE 1: VERIFY EVERYTHING

### Definition:

**"Always verify, never trust"**

### What Gets Verified:

- Every user must authenticate
- Every device must be checked
- Every request is validated

### Implementation:

- Multi-factor authentication
- Device compliance checks
- Continuous identity verification

---

## 🔵 PRINCIPLE 2: LEAST PRIVILEGE ACCESS

### Definition:

**"Give only minimum access required"**

### What It Means:

- User gets access only to needed resources
- No extra permissions
- Limited to specific functions

### Real-World Example:

- **Student:** Only college portal access
- **Manager:** Access to team files
- **Admin:** System configuration access

---

## 🟢 PRINCIPLE 3: INSPECT & MONITOR ALL TRAFFIC

### Definition:

**"Continuous visibility and monitoring"**

### What Gets Monitored:

- Even allowed traffic is checked
- User behavior analyzed
- Network activity logged
- Anomalies detected

---

## 🧠 MEMORY TRICK

### "VLM = Verify → Least Privilege → Monitor"

1. **V**erify identity
2. **L**imit access
3. **M**onitor activity

---

## 🧠 QUICK SUMMARY

✔ **Verify Everything**

✔ **Three pillars of Zero Trust**
