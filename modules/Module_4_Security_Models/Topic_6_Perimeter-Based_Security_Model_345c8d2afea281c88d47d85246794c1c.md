# Topic 6: Perimeter-Based Security Model

> Traditional Boundary-Based Security Approach
> 

---

## 📖 SIMPLE EXPLANATION

**Also called: "Castle and Moat Model"**

🔹 Strong security at boundary (firewall)

🔹 Inside network = trusted

🔹 Outside network = untrusted

---

## 🧠 CORE IDEA

**Protect only the network boundary**

- Outside = untrusted ❌
- Inside = trusted ✅
- Firewall = castle walls

---

## 🌍 REAL-WORLD ANALOGY

### Think of a Castle 🏰

- Strong walls protect from outside
- Inside people are trusted
- No internal checks

---

## ❗ PROBLEM WITH THIS MODEL

### Today's World:

- Remote work
- Cloud computing
- Mobile devices
- Boundary is no longer clear

**Result:** Model is outdated

---

## ⚠️ MAJOR LIMITATIONS

### 🔴 1. TRUSTING INTERNAL NETWORK

If attacker enters:

- Full access inside ❌
- Can move freely
- No verification

### 🟡 2. CANNOT HANDLE MODERN ACCESS

- Work from home (WFH)
- Cloud services
- Mobile devices
- No perimeter to protect

### 🔵 3. INSIDER THREATS

- Employees can misuse access
- No internal monitoring
- Lateral movement possible

### 🟢 4. POOR VISIBILITY

Cannot detect:

- Encrypted traffic
- User identity properly
- Behavioral anomalies

---

## ⚙️ HOW IT WORKS (STEP-BY-STEP)

### Step 1: Firewall at Boundary

- Filter external traffic

### Step 2: Block External Threats

- Stop bad traffic

### Step 3: Allow Internal

- Trust internal communication
- No further checking

---

## ⚠️ EXAM TRAP

### ❌ Wrong Thinking:

"Perimeter security = enough"

### ✅ Correct Understanding:

"It is outdated alone → needs Zero Trust"

---

## 💡 MEMORY TRICK

### "Strong outside, weak inside"

---

## 🧠 QUICK SUMMARY

✔ Boundary-based security

✔ Internal network trusted

✔ Fails in modern environment

✔ Replaced by Zero Trust

---

## 📝 INTERVIEW TIP

### Question:

**"Why is perimeter security not enough today?"**

### Answer:

"Because modern environments (cloud, remote work) remove clear network boundaries and attackers can bypass perimeter defenses."
