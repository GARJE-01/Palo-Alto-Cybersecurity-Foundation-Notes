# Topic 7: Security Policy (Subjects, Objects, Actions)

> Rules That Define Who Can Do What on Which Resources
> 

---

## 📖 SIMPLE EXPLANATION

**Security Policy = Set of rules defining who can do what on which resources**

👉 Defines system security rules

👉 Prevents unauthorized access

👉 Controls user behavior

---

## 🧠 CORE COMPONENTS (VERY IMPORTANT)

### 🔴 1. SUBJECTS

**Who is accessing the system**

- Users
- Admins
- Applications
- Attackers

### 🔵 2. OBJECTS

**What is being accessed**

- Files
- Databases
- Servers
- Applications

### 🔶 3. ACTIONS

**What operation is performed**

- Read
- Write
- Delete
- Execute

### 🔷 4. PERMISSIONS

**What is allowed or denied**

- Read-only
- Full access
- No access

### 🔺 5. PROTECTIONS

**Security mechanisms used**

- Encryption
- Authentication
- Access control

---

## ⚙️ HOW IT WORKS (STEP-BY-STEP)

1. **Subject requests access**
2. **System checks policy**
3. **Permissions verified**
4. **Action allowed/denied**

---

## 🌍 REAL-WORLD EXAMPLE

### College System:

- **Subject:** Student
- **Object:** Result database
- **Action:** Read
- **Permission:** Allowed ✅

Student cannot edit marks ❌

---

## ⚠️ EXAM TRAP

### ❌ Wrong:

"Subject vs Object confusion"

### ✅ Right:

**Subject = user | Object = resource**
