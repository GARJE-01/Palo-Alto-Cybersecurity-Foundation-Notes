# Topic 8: Security Models (DAC, MAC)

> Frameworks for Access Control Implementation
> 

---

## 📖 SIMPLE EXPLANATION

**Security Models = frameworks defining how access control is implemented**

Two main types:

- DAC (Discretionary)
- MAC (Mandatory)

---

## 🔴 DISCRETIONARY ACCESS CONTROL (DAC)

### Definition:

**Owner decides access**

- User controls their files
- Can give permissions to others
- Flexible

### Advantages:

✅ Easy to use

✅ Flexible

### Disadvantages:

❌ Less secure

❌ Users may give wrong permissions

### Example:

You create a file → decide who reads/edits

---

## 🔵 MANDATORY ACCESS CONTROL (MAC)

### Definition:

**System decides access**

- Controlled by central authority
- Users cannot change permissions
- Strict

### Advantages:

✅ Highly secure

✅ Strict control

### Disadvantages:

❌ Less flexible

❌ Hard to manage

### Example:

Military system (Confidential, Secret, Top Secret)

---

## 📊 DAC vs MAC COMPARISON

| Feature | DAC | MAC |
| --- | --- | --- |
| **Control** | User | System/Admin |
| **Flexibility** | High | Low |
| **Security** | Lower | Higher |

---

## ⚠️ EXAM TRAP

### Simple Rule:

- **DAC = User decides**
- **MAC = System decides**

---

## 🧠 MEMORY TRICK

### "D = Decide by user, M = Managed by system"

---

## 🧠 QUICK SUMMARY

✔ DAC → flexible, user-controlled

✔ MAC → secure, system-controlled

✔ Different use cases
