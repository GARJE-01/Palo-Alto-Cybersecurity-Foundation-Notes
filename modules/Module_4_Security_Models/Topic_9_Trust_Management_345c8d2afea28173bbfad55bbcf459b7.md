# Topic 9: Trust Management

> System for Defining and Enforcing Security Rules
> 

---

## 📖 SIMPLE EXPLANATION

**Trust Management = System that defines and enforces security rules using logic**

👉 Uses formal language

👉 Ensures policies followed

👉 Automated enforcement

---

## 🧠 CORE COMPONENTS

### 🔴 1. POLICY LANGUAGE

**Used to define rules**

- Who can access what
- What actions allowed
- Formal syntax

### 🔵 2. COMPLIANCE CHECKER

**Enforces rules**

- Checks system follows policy
- Allows/denies actions
- Automated verification

---

## 🔑 ADDITIONAL ELEMENTS

- **Principals:** users or systems
- **Actions:** operations performed
- **Policies:** rules
- **Credentials:** proof of identity

---

## ⚙️ HOW IT WORKS (STEP-BY-STEP)

1. Define policy rules
2. User requests access
3. System checks credentials
4. Compliance checker verifies
5. Access granted or denied

---

## 🌍 REAL-WORLD EXAMPLE

### Company System:

1. Employee requests access
2. System checks:
    - Role
    - Credentials
3. Policy decides access

---

## ❓ WHY IT MATTERS

✅ Ensures consistent security

✅ Automates policy enforcement

✅ Reduces human error

---

## ⚠️ EXAM TRAP

### ❌ Wrong:

"Trust = Trust Management"

### ✅ Right:

- **Trust** = belief
- **Trust Management** = system enforcing rules
