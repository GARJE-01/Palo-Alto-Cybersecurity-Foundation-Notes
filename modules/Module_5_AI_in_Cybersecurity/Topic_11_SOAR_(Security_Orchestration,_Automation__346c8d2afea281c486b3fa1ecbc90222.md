# Topic 11: SOAR (Security Orchestration, Automation & Response)

> Platform that automates security operations by connecting and automating tools
> 

---

## 📖 SIMPLE EXPLANATION

**SOAR** = Platform that automates security operations

It combines:

- Multiple tools
- Multiple processes
- Automation

📌 **SOAR integrates and automates multiple security tools and processes**

---

## 🧠 BREAKING DOWN SOAR

### 🔴 1. ORCHESTRATION

**Connects different security tools together**

Example tools:

- Firewall
- SIEM (Security Information Event Management)
- Antivirus
- Endpoint Detection
- Cloud security

✔ Makes tools work together

### 🔵 2. AUTOMATION

**Performs tasks automatically**

Example tasks:

- Alert handling
- Threat blocking
- Playbook execution

✔ Reduces manual work

### 🟡 3. RESPONSE

**Takes action against threats**

Example actions:

- Isolate infected system
- Block IP address
- Quarantine malware
- Notify security team
- Create tickets

✔ Stops threats faster

---

## ⚙️ HOW SOAR WORKS (STEP-BY-STEP)

1. **Alert generated** from security tool
2. **SOAR collects data** from multiple sources
3. **Analyzes threat** using AI/ML
4. **Executes response automatically** based on rules
5. **Reports incident** to team

---

## 🌍 REAL-WORLD EXAMPLE

Scenario: **Suspicious login detected**

SOAR:

- 🌟 Blocks account automatically
- 🌟 Sends alert to security team
- 🌟 Logs activity for review
- 🌟 Starts investigation playbook

---

## 🎯 WHY IT MATTERS

✔ **Reduces manual work** - less human effort

✔ **Faster response** - seconds instead of hours

✔ **Minimizes damage** - acts immediately

✔ **Consistent** - always follows rules

✔ **24/7 operation** - never sleeps

---

## ⚠️ EXAM TRAP

**SOAR vs SIEM:**

| Tool | Role |
| --- | --- |
| **SIEM** | **Detects** threats |
| **SOAR** | **Responds** to threats |

📌 **SIEM finds it, SOAR fixes it**
