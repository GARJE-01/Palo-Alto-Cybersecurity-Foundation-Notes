# Topic 7: Machine Learning in Cybersecurity

> ML = AI technique where systems learn from data and improve automatically
> 

---

## 📖 SIMPLE EXPLANATION

**Machine Learning (ML)** = AI technique where systems learn from data and improve automatically

📌 **From your module: ML learns from past data and predicts future outcomes**

---

## 🧠 HOW ML WORKS

ML uses:

- Past data
- Patterns in that data
- Probabilities

Then:

- ✔ Predicts future threats
- ✔ Classifies new threats
- ✔ Improves automatically

📌 **ML uses probabilities and previous outcomes for prediction**

---

## 🚨 THE PROBLEM (Why ML is needed)

Traditional systems:

- Static rules (don't change)
- Cannot detect new threats
- Miss unknown attack types
- Require manual updates

---

## 💡 ML SOLUTION

ML:

- ✔ **Detects unknown attacks** (never seen before)
- ✔ **Finds hidden patterns** automatically
- ✔ **Works faster than humans**
- ✔ **Improves continuously**

📌 **ML detects anomalies and improves threat classification**

---

## 🔴 5 KEY APPLICATIONS OF ML

### 🔴 1. MALWARE DETECTION

- Identifies malicious files
- Analyzes file behavior
- Classifies threats

### 🔵 2. ANOMALY DETECTION

- Detects unusual behavior
- Finds outliers
- Spots abnormal patterns

### 🟡 3. USER BEHAVIOR ANALYSIS

- Tracks suspicious activity
- Monitors user patterns
- Detects insider threats

### 🟢 4. VULNERABILITY PRIORITIZATION

- Identifies critical risks
- Ranks vulnerabilities
- Suggests patches

### 🟣 5. THREAT PREDICTION

- Predicts future attacks
- Risk scoring
- Threat forecasting

📌 **ML performs malware detection, anomaly detection, and threat prediction**

---

## ⚠️ REQUIREMENTS FOR ML

**ML needs:**

- ✔ Large amount of data
- ✔ High-quality data
- ✔ Processing power
- ✔ Defined algorithms

📌 **ML requires large, high-quality, and structured data**

---

## ⚠️ LIMITATIONS OF ML

### 🔴 1. DATA DEPENDENCY

- Poor data = wrong results
- Biased data = biased predictions

### 🟡 2. HIGH RESOURCE USAGE

- Needs strong hardware
- Expensive computations

### 🔵 3. ERROR POSSIBILITY

- Not always accurate
- Can make mistakes

📌 **ML has error susceptibility and depends on data quality**

---
