# Topic 2: 4 Key Capabilities

> Four essential capabilities that define the platform
> 

---

## 🔥 THE 4 CAPABILITIES (EXAM CRITICAL)

---

## 🟢 1. FULL VISIBILITY

### What You Can See:

- 👤 **Who** (user)
- 📱 **What** (application)
- 📍 **Where** (device/location)

### Across:

- 🔗 Network
- ☁️ Cloud
- 💻 Endpoints

### Example:

- User: John
- Application: Gmail
- Device: Laptop from Mumbai

---

## 🔴 2. REDUCE ATTACK SURFACE

### Goal:

Minimize entry points for attackers

### Using:

- 🔐 Access control
- 🧩 Network segmentation
- ❌ Zero Trust principles

### How:

- Only required access allowed
- Everything else blocked

---

## 🔵 3. PREVENT KNOWN THREATS

### Block:

- 🦠 Malware
- 🔐 Ransomware
- 🤖 Botnets
- 💣 Exploits

### Uses:

- 📊 Threat intelligence
- 🔍 Signature-based detection
- 🛡️ Advanced threat prevention

---

## 🟡 4. DETECT UNKNOWN THREATS

### Uses:

- 🤖 AI/ML
- ⚙️ Automation
- 📊 Behavioral analysis

### Detects:

- 💥 Zero-day attacks
- 🎭 Unusual patterns
- 🔍 Advanced threats

---

## 📊 CAPABILITY FLOW

```
Visibility → Reduce Surface → Prevent Known → Detect Unknown
```

---

## 🧠 MEMORY TRICK

**"See → Limit → Block → Catch"**

---

## 🎯 QUICK SUMMARY

✅ Full visibility (who, what, where)

✅ Reduce attack surface (limit access)

✅ Prevent known threats (block malware)

✅ Detect unknown threats (catch zero-days)

✅ Four pillars of defense
