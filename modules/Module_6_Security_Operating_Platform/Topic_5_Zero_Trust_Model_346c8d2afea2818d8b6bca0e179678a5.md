# Topic 5: Zero Trust Model

> Trust NO ONE - Verify EVERYTHING
> 

---

## 🧠 SIMPLE EXPLANATION

**Zero Trust** = Trust nothing, verify everything

### Core Principle:

**"Never trust, always verify"**

---

## ❌ OLD MODEL vs ✅ ZERO TRUST

### OLD Model:

❌ "Trust internal users"

❌ "Inside network = safe"

❌ Basic access control

### ZERO TRUST Model:

✅ "Trust NO ONE"

✅ "Verify everyone"

✅ "Check everything"

---

## 🔑 ZERO TRUST PRINCIPLES

### 1. Verify Every Request

- Every user must authenticate
- Every device must be checked
- Every request is inspected

### 2. Least Privilege Access

- Only minimum access needed
- Everything else is blocked
- Access is constantly verified

### 3. Inspect All Traffic

- No implicit trust
- All communication monitored
- Threats are detected instantly

---

## 🌍 REAL-WORLD EXAMPLE

### Scenario: Employee accessing files

**With Zero Trust:**

1. ✅ User must authenticate (password + MFA)
2. ✅ Device must be secure (not infected)
3. ✅ Only needed files accessible (least privilege)
4. ✅ Activity is monitored (all traffic inspected)
5. ✅ Suspicious behavior = instant block

---

## 💡 KEY CONCEPT

**Even inside company network:**

- User must authenticate
- Access is limited
- Everything is monitored

**"Nothing is trusted by default"**

---

## 🧱 ZERO TRUST ARCHITECTURE COMPONENTS

### 1. Identity Verification

- MFA (Multi-Factor Authentication)
- Device authentication
- Behavioral analysis

### 2. Access Control

- Least privilege assignment
- Microsegmentation
- Continuous verification

### 3. Monitoring

- All traffic inspected
- Behavior analyzed
- Threats detected

---

## 🧠 EXAM TRAP

**WRONG:** "Zero Trust = only for external users"

**RIGHT:** "Zero Trust = applied to EVERYONE and EVERYTHING"
