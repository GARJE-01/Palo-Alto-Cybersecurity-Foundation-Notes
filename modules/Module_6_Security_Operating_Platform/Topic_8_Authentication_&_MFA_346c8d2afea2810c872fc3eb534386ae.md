# Topic 8: Authentication & MFA

> Verifying user identity with strong security
> 

---

## SIMPLE EXPLANATION

**Authentication** = Verifying who the user is

**"Are you really who you say you are?"**

---

## TYPES OF AUTHENTICATION

### SINGLE FACTOR AUTHENTICATION (SFA)

- Password only
- Weak security
- Passwords can be stolen

### MULTI-FACTOR AUTHENTICATION (MFA / 2FA)

- Password + OTP (One-Time Password)
- Password + Mobile verification
- Password + Biometric

---

## MFA FACTORS

### Something You Know:

- Password, PIN, Security question

### Something You Have:

- Phone, Security key, Hardware token

### Something You Are:

- Fingerprint, Face recognition, Iris scan

---

## REAL-WORLD EXAMPLE

### Without MFA:

- Type password
- Hacker types same password
- Access granted to hacker!

### With MFA:

- Type password
- Hacker types password
- System asks for OTP
- Hacker doesn't have phone
- Access denied!

---

## WHY MFA MATTERS

Passwords get compromised

MFA is hard to bypass

Protects sensitive data

Required by compliance

Recommended best practice

---

## MEMORY TRICK

**"Two factors = Twice as secure"**

---

## QUICK SUMMARY

Authentication verifies user

SFA (password) = weak

MFA (password + something) = strong

Use MFA always

Greatly improves security
